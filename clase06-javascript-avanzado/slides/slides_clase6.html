🤔 EL PROBLEMA: Código Repetitivo

Tu código de la Clase 5 probablemente se ve así:

// ❌ Código repetitivo y sin decisiones let proyectos = ["Portfolio", "Blog", "Tienda"]; for (let i = 0; i < proyectos.length; i++) { console.log(proyectos[i]); } // Más código repetitivo... for (let i = 0; i < proyectos.length; i++) { console.log(`Proyecto: ${proyectos[i]}`); } // Sin validaciones, sin decisiones, sin organización

Problemas: Se repite código, no hay validaciones, no toma decisiones, difícil de mantener

✨ LA SOLUCIÓN: Código Inteligente

Al final de esta clase, tu código se verá así:

// ✅ Código organizado e inteligente function validarProyecto(proyecto) { return proyecto.length > 0 ? "Válido" : "Inválido"; } function mostrarProyectos(lista) { lista.forEach(proyecto => { const estado = validarProyecto(proyecto); console.log(`${proyecto}: ${estado}`); }); } const proyectosValidos = proyectos.filter(p => p.length > 0); const datosJSON = JSON.stringify(proyectosValidos, null, 2);

¡Nivel Up! Funciones reutilizables, decisiones automáticas, datos listos para APIs

🚦 DEMO: Condicionales

Archivo: ejemplo-1-condicionales.html

Las condicionales permiten que tu código tome decisiones

// Ejemplo básico de decisión let edad = 18; if (edad >= 18) { console.log("✅ Puedes votar"); } else { console.log("❌ Aún no puedes votar"); } // Para tu proyecto let proyecto = { nombre: "Portfolio", completado: true }; if (proyecto.completado) { console.log(`🎉 ${proyecto.nombre} está listo!`); } else { console.log(`🔧 ${proyecto.nombre} en desarrollo`); }

🎯 Tu código ahora puede "pensar"

⚖️ OPERADORES: Las Reglas del Juego

🔍 Comparación:

=== // Igual exacto !== // Diferente > // Mayor que < // Menor que >= // Mayor o igual <= // Menor o igual

🧠 Lógicos:

&& // AND (y) || // OR (o) ! // NOT (no) // Ejemplos: edad >= 18 && tieneLicencia precio < 100 || esOferta

💡 Regla dorada: Siempre usa === en lugar de ==

🔀 SWITCH y OPERADOR TERNARIO

🔀 Switch para múltiples opciones:

switch (dia) { case "lunes": mensaje = "Inicio de semana"; break; case "viernes": mensaje = "¡Por fin viernes!"; break; default: mensaje = "Día normal"; }

❓ Ternario para decisiones rápidas:

// Sintaxis: condición ? true : false let estado = completado ? "Listo" : "Pendiente"; let precio = esMiembro ? 50 : 100; let mensaje = edad >= 18 ? "Adulto" : "Menor";

🔧 DEMO: Funciones

Archivo: ejemplo-2-funciones.html

Las funciones organizan tu código y lo hacen reutilizable

// Sin función: código repetitivo ❌ console.log(`¡Hola Ana!`); console.log(`¡Hola Luis!`); console.log(`¡Hola María!`); // Con función: código organizado ✅ function saludar(nombre) { return `¡Hola ${nombre}!`; } console.log(saludar("Ana")); console.log(saludar("Luis")); console.log(saludar("María"));

🎯 Una función = una responsabilidad específica

🏗️ ANATOMÍA DE UNA FUNCIÓN

function calcularDescuento(precio, porcentaje) { let descuento = precio * (porcentaje / 100); let precioFinal = precio - descuento; return precioFinal; } // Usar la función let precioOferta = calcularDescuento(100, 20); // 80
  • function: palabra clave para crear función
  • calcularDescuento: nombre descriptivo
  • (precio, porcentaje): parámetros de entrada
  • return: valor que devuelve la función

🏹 ARROW FUNCTIONS: Sintaxis Moderna

📝 Función tradicional:

function duplicar(numero) { return numero * 2; } function saludar(nombre) { return `¡Hola ${nombre}!`; }

✨ Arrow function:

const duplicar = numero => numero * 2; const saludar = nombre => `¡Hola ${nombre}!`; // Con múltiples líneas const calcular = (a, b) => { let suma = a + b; return suma * 2; };

💡 Arrow functions: más cortas para funciones simples

🎯 DEMO: Callbacks

Archivo: ejemplo-5-callbacks.html

¡Llegó el momento! Métodos modernos de arrays

🏚️ Forma antigua (Clase 5):

let proyectos = ["Portfolio", "Blog"]; for (let i = 0; i < proyectos.length; i++) { console.log(proyectos[i]); }

✨ Forma moderna (Clase 6):

let proyectos = ["Portfolio", "Blog"]; proyectos.forEach(proyecto => { console.log(proyecto); });

🎯 Callbacks = funciones que se pasan como parámetros

🎭 LOS TRES MOSQUETEROS de Arrays

let numeros = [1, 2, 3, 4, 5]; // 🔄 forEach: Hacer algo con cada elemento numeros.forEach(num => console.log(num * 2)); // 🔄 map: Transformar y crear nuevo array let dobles = numeros.map(num => num * 2); console.log(dobles); // [2, 4, 6, 8, 10] // 🔍 filter: Seleccionar elementos específicos let pares = numeros.filter(num => num % 2 === 0); console.log(pares); // [2, 4]
  • forEach: Ejecuta algo, NO devuelve nada
  • map: Transforma elementos, devuelve nuevo array
  • filter: Selecciona elementos, devuelve nuevo array
